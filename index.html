真相

兇手：HttpClientHandler

犯案手法：未正確將 TLS 版本和憑證驗證設定套用到實際的 HTTP Request

破案關鍵：透過命令列手動測試，發現跳過憑證檢查可成功，進而鎖定 TLS/SSL 層問題。最終發現 HttpClientHandler 的實作缺陷，改用 HttpWebRequest 解決。

啟示：

  • 即使程式碼 3 年沒動，環境變化（憑證過期/更新）也可能導致問題
  • 封測與正式的差異不只是 config，OS/runtime 行為也可能不同
  • 有時候 framework 的抽象層反而會隱藏問題
  • 系統性排查法（逐層排除）是解決未知問題的最佳策略
